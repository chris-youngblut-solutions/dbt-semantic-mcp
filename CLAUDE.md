# dbt-semantic-mcp

DuckDB + dbt medallion marts + a governed semantic layer (MetricFlow), exposed to agents over MCP

**Tier**: T1  **Language**: python  **Test discipline**: pragmatic

This file is **read by every Claude Code session** that opens this repo.
Keep it short, scannable, and load-bearing. Long-form prose belongs in
`docs/`. Personal scratch notes (TODOs, ad-hoc commands, half-formed

## What this is

A demo warehouse stack: synthetic seeds → dbt medallion models → MetricFlow
semantic layer → MCP server (stdio) answering metric queries from the governed
definitions. It does NOT compose SQL from user input, ship a cloud-warehouse
config, or run an orchestrator — see README Limits.

## Build / test / lint

All verbs go through `just`. Never run the underlying tool directly in
docs — that creates two sources of truth.

```sh
just            # list all recipes
just fmt        # auto-format
just lint       # lint (warnings → errors)
just test       # run tests per discipline (pragmatic)
just build      # build release artifact
just check      # fmt + lint + test (the merge gate)
```

Tier-2 release path:

```sh
just release patch    # tag + push (CI does the build/sign/SBOM/SLSA)
just release minor
just release major
```

## Architecture

- `warehouse/` — the dbt project (DuckDB; profiles.yml in-project). Run dbt/mf
  commands from this directory.
- `warehouse/seeds/*.csv` — generated by `scripts/generate_seed.py`, committed;
  a test enforces committed == regenerated. Never hand-edit a seed CSV.
- `src/dbt_semantic_mcp/` — MCP server; reads only dbt/MetricFlow artifacts
  (semantic_manifest.json, manifest.json, the `mf` CLI). Keep it that way —
  no SQL composed from input.
- `tests/` — pragmatic discipline; the conftest builds the warehouse if
  target/ is missing.
- `.github/workflows/` — CI; SHA-pinned actions; runs dbt build +
  mf validate-configs before pytest.
